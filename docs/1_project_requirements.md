# Project Requirements

## Building the Data Warehouse (Data Engineering)

### Objective

Develop a **modern data warehouse using SQL Server** to consolidate sales data from multiple systems.  
The goal is to enable **analytical reporting and support data-driven decision-making**.

---

## Specifications

### Data Sources
- Import data from **two source systems**:
  - **ERP**
  - **CRM**
- Data is provided as **CSV files**.

---

### Data Quality
- Identify and resolve **data quality issues** before analysis.
- Apply data cleansing techniques such as:
  - Removing duplicates
  - Handling NULL values
  - Standardizing categorical values
  - Fixing invalid dates or inconsistent data

---

### Data Integration
- Combine data from **ERP and CRM systems**.
- Create a **single, user-friendly data model** optimized for analytical queries.

---

### Scope
- The project focuses on the **latest available dataset only**.
- **Historical data tracking (historization)** is **not required**.

---

### Documentation
- Provide **clear and structured documentation** of the data model.
- Documentation should support:
  - **Business stakeholders**
  - **Analytics teams**
  - **Future data engineering development**

---

## Expected Outcome

The final solution should deliver:

- A **structured data warehouse architecture**
- **Clean and integrated datasets**
- A **business-ready analytical data model**
- Documentation to support **data understanding and usage**
